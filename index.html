2.4 Firebase & Cloud Data

New: We use Firebase for:
  • User Authentication: Anonymous user ID generated for each device
  • Cloud Storage: Generated content synced to cloud (optional)
  • Usage Tracking: Daily generation limits and PRO status
  • Analytics: User behavior, feature popularity, retention metrics
  • Crash Reporting: Firebase Crashlytics for bug fixes
  • Push Notifications: FCM tokens for sending notifications about trending content
  • Remote Configuration: App settings managed remotely

2.5 Third-Party AI Services

Updated: Data sent to AI providers:
  • Google Gemini AI: Your input topics, platform selection, language preference
  • Groq API: Alternative AI processing for content generation
  • Data Sent: Topic + Platform + Language + Content Type to generate viral content
  • Note: Do not enter sensitive personal information in topics

2.6 Advertising Data (AdMob)

  • Advertising ID: Device's GAID for personalized ads
  • Ad Interactions: Cicks, views, engagement metrics
  • Ad Preferences: User ad personalization settings
  • Demographics: Approximate age/gender for ad targeting (if available)
